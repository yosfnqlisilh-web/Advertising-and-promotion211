# توثيق مشروع "فن الإعلان"

## 1. نظرة عامة على المشروع

هذا المستند هو المرجع الرسمي لموقع "فن الإعلان"، وهو تطبيق ويب حديث تم تطويره باستخدام Next.js ونشره على منصة Firebase App Hosting. تم تصميم الموقع ليكون واجهة رقمية احترافية للشركة، مع التركيز على عرض الخدمات، والأعمال السابقة، وتسهيل التواصل مع العملاء عبر نموذج اتصال ومساعد ذكي (شات بوت).

---

## 2. التصميم والميزات

### الهوية البصرية
- **اللون الرئيسي:** تم استخدام اللون البرتقالي (`#ff9000`) المستوحى من شعار المؤسسة كلون أساسي لإبراز العناصر التفاعلية والهامة.
- **الخطوط:** تم اعتماد خط `Noto Kufi Arabic` لضمان تجربة قراءة مريحة وواضحة باللغة العربية.
- **النمط العام:** تصميم عصري ونظيف يستخدم تأثيرات الظل والتأثيرات الزجاجية (Glassmorphism) لإضفاء لمسة من الفخامة والعمق.

### الميزات الرئيسية
- **صفحات متعددة:** (الرئيسية، معرض الأعمال، الخدمات، المدونة، اتصل بنا).
- **تصميم متجاوب:** الموقع يعمل بشكل مثالي على جميع الأجهزة (الجوال، التابلت، سطح المكتب).
- **معرض أعمال ديناميكي:** يعرض صور المشاريع مع تأثيرات تفاعلية.
- **مساعد ذكي (شات بوت):**
    - شخصية بوت تفاعلية (حركة العين، تعابير الوجه).
    - يرد على استفسارات العملاء بناءً على قاعدة معرفية محددة.
    - يقدم خيارات للتواصل المباشر (واتساب، خرائط جوجل).
- **نموذج "اتصل بنا":** يعمل باستخدام Server Actions لإرسال البيانات بشكل آمن وفعال.
- **تحسين محركات البحث (SEO):** تم بناء الموقع مع مراعاة أفضل ممارسات SEO.

---

## 3. البنية التقنية

- **إطار العمل:** Next.js 14 (باستخدام App Router).
- **الاستضافة:** Firebase App Hosting (استضافة ديناميكية قائمة على الخوادم).
    - يتم تشغيل منطق الخادم (Server-side rendering) عبر دالة سحابية (Cloud Function) مُدارة تلقائياً.
- **التصميم:** Tailwind CSS.
- **التفاعل:** React Server Components و Client Components.
- **النماذج:** Next.js Server Actions.
- **الشات بوت:** Google Gemini API (يتم استدعاؤه من خلال Server Action).

## 5. إدارة المحتوى و SEO

### خريطة الموقع (Sitemap)
- **الغرض:** لفهرسة الموقع في محركات البحث مثل جوجل.
- **الرابط:** **[https://advertising-and-promotion211.web.app/sitemap.xml](https://advertising-and-promotion211.web.app/sitemap.xml)**

### الكلمات البحثية والمحتوى (SEO)
- لا يوجد ملف واحد للكلمات البحثية، بل تم دمج المحتوى والبيانات الوصفية (metadata) مباشرة في كل صفحة لتحقيق أفضل النتائج.
- **لمراجعة وتعديل المحتوى:**
    - **بيانات الصفحات الوصفية (Title, Description):** موجودة في ملفات `layout.tsx` و `page.tsx` لكل صفحة. (مثال: `app/page.tsx`, `app/services/page.tsx`).
    - **محتوى المدونة:** المقالات موجودة كملفات Markdown في مجلد `src/markdown/blogs`. يمكنك تعديلها أو إضافة مقالات جديدة هناك.
    - **قاعدة معرفة الشات بوت:** النصوص المثبتة التي يجيب بها البوت موجودة في الملف `src/app/actions.ts` داخل دالة `getGeminiResponse`.
